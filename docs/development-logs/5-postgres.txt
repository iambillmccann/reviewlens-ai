Model details implemented:

  - id UUID primary key
  - auth_provider nullable string

  - email nullable unique string
  - display_name nullable string
  - avatar_url nullable string
  - created_at timestamp
  - updated_at timestamp
- workspaces:
  - id UUID primary key
  - name string
  - created_at timestamp
  - updated_at timestamp
- workspace_members:
  - id UUID primary key
  - workspace_id FK -> workspaces.id
  - user_id FK -> users.id
  - role string default owner
  - created_at timestamp
  - updated_at timestamp
  - unique(workspace_id, user_id)

7) Service layer behavior
- Added apps/api/app/services/user_bootstrap.py:
  - get_or_create_local_user_with_default_workspace(...)
- Input fields:
  - auth_provider
  - auth_provider_user_id
  - email
  - display_name
  - avatar_url
- Lookup precedence:
  1. auth_provider + auth_provider_user_id
  2. email fallback
- Behavior:
  - Return existing user when found.
  - If not found, create user + default workspace + workspace_members owner row in one transaction.
- Workspace naming rule:
  - "<display_name>'s Workspace" when display_name exists
  - otherwise "My Workspace"
