MILESTONE: CI Optimization (Path Filtering)
Date: May 13, 2026
Status: Complete

GOAL
Reduce CI noise by only running relevant checks for changed files.

IMPLEMENTATION

1. Path-aware CI job filtering
   - Added `detect-changes` job using `dorny/paths-filter@v1`
   - Filters for `apps/web/**` changes → triggers `web-checks` only
   - Filters for `apps/api/**` changes → triggers `api-checks` only
   - CI workflow file changes (`*.yml`) trigger both checks
   - Always run both checks on pull requests (safe default)

2. Behavior
   - Documentation-only changes: no CI jobs run
   - Frontend-only changes: `web-checks` runs (lint, typecheck, build)
   - Backend-only changes: `api-checks` runs (lint, tests)
   - Mixed or workflow changes: both jobs run

3. Benefits
   - Faster feedback for single-app changes
   - Reduced CI resource usage for documentation/config updates
   - Monorepo-aware without needing to change how we structure code

VALIDATION

This milestone is tested by pushing documentation changes only; CI should skip both checks.
